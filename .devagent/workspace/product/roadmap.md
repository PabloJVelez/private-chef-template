# Product Roadmap

Milestones that translate mission themes into delivery. Focus: fast client onboarding (2-week demo, ~1-month first mock order) and a living template that improves with each chef.

## Horizon 1 — Foundation (current)
- Stable Medusa 2 backend + Remix storefront monorepo
- Stripe payment integration, product/category/collection support
- Developer-friendly setup (TypeScript, Biome, env generation, Docker)
- Core concepts and patterns proven with first and second chef clients

## Horizon 2 — Repeatable client onboarding
- **Templatized surface:** Clearly defined values to swap per chef (branding, copy, products, config) and a repeatable “plug in the chef” flow.
- **Delivery cadence:** Documented/automated path to dev demo in 2 weeks and first mock order ~1 month from interest.
- **Living template:** Process to decide what from each client gets folded back (features, UX patterns, content structure) so the next client starts from a better base.

## Horizon 3 — Scale and differentiation
- Richer shared feature set from multiple chefs (e.g. booking, menus, availability, chef-specific UX).
- Optional: lightweight “client onboarding checklist” or runbook so the 2-week / 1-month targets are easy to hit and track.
